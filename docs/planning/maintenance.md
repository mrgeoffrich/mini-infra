# Maintenance

Catch-all project for one-off maintenance and follow-up work in Mini Infra.

Tickets are tracked in Linear under the **Maintenance** project on the Altitude
Devops team. Each ticket is independent — they do not block each other and do
not have to ship in any particular order.

This doc is intentionally a stub. The ticket body in Linear (Goal /
Deliverables / Done when / Relevant docs / Smoke tests) is the contract for
each task — there is no per-task design here.

Use the `task-to-linear` skill to file new maintenance tickets, and
`execute-next-task` (with no argument, or with an explicit `ALT-NN`) to pick
one up and ship it.
